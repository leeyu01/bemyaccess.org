  Key principles:
  - Separation of concerns: Keep the Python crawler in its own directory at the project root, not inside the Next.js app
  - Independent development: Each can be developed, tested, and deployed independently
  - Clear boundaries: They communicate only via API calls (Next.js API + production URLs)
  - Separate dependency management: package.json for Next.js, requirements.txt for Python
  - Environment separation: Different .env files for each component

  This structure makes it easy to:
  - Develop both locally simultaneously
  - Deploy independently (Vercel for Next.js, internal VMs for crawler)
  - Keep codebases clean and maintainable
  - Avoid mixing language ecosystems
